# ── PyTorch (CPU) ─────────────────────────────────────────────────────────────
# H-13 FIX: torch was absent from requirements.txt — only installed by the
# Dockerfile explicitly.  Any pip install -r requirements.txt path (CI, local
# dev, fresh venv) produced an environment without torch, causing
# long_term_memory._embed_cpu() to silently skip device management and hit
# GPU contention on the RTX 3050 4GB.
# CPU-only wheel keeps the install lightweight for non-GPU environments.
# The Dockerfile overrides this with the pinned torch==2.3.1+cpu wheel from
# the PyTorch index, so production image is unaffected.
# P3-14 FIX: added <2.4.0 upper bound.  Without it, `pip install -r
# requirements.txt` on a developer machine would resolve the latest PyTorch
# (currently 2.6+), which has breaking changes to torch.cuda device
# management APIs used by long_term_memory._embed_cpu().  The Dockerfile is
# pinned to torch==2.3.1+cpu; the upper bound here keeps local dev consistent
# with the production image.  Bump both together when upgrading PyTorch.
torch>=2.3.1,<2.4.0
